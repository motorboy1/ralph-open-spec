# Decision Rationale (EXTENDED) v1 — Why RALPH Core + Vibe Studio Shell

## Decision
- Governance stays in RALPH.
- Workspace UI lives in Vibe Studio.

## Rationale
1) Durability: rules outlive UI churn
2) Accountability: DONE requires evidence
3) Portability: swap models/tools without rewriting governance
4) Collaboration: share packages without sharing entire runtime

## Implication
- Vibe Studio implements activation & visualization layers.
- RALPH enforces correctness, continuity, and reproducibility.
